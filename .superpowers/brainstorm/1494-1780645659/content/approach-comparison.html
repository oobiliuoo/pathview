技术方案对比

三种前后端分离架构，本地/单机部署

A

React + Three.js + Express + SQLite

前端: React + Three.js (react-three-fiber) 构建3D视图

后端: Express + SQLite 轻量持久化

优点: 生态成熟、社区资源丰富、SQLite零配置

缺点: 前端包体积较大、React对纯展示页略显重量级

B

Vue3 + Three.js + FastAPI + SQLite

前端: Vue3 + Three.js (troisjs) 构建3D视图

后端: Python FastAPI + SQLite

优点: Python生态好、FastAPI性能高且自带API文档、Vue3轻量

缺点: 需要Python环境、前后端语言栈不同

C

Vue3 + Three.js + Express + SQLite

前端: Vue3 + Three.js (troisjs) 构建3D视图

后端: Express + SQLite

优点: 全JS栈、Vue3比React轻量、troisjs与Vue集成好

缺点: troisjs社区比react-three-fiber小

推荐：方案 B (Vue3 + Three.js + FastAPI + SQLite)

推荐理由

  • Python后端在工业/机器人领域更常见，后续扩展方便
  • FastAPI自动生成API文档，调试方便
  • Vue3 + Vite 开发体验流畅
  • SQLite单文件数据库，本地部署零配置

需考虑

  • 需要Python环境（可用uv管理）
  • 前后端语言栈不同
